# 06 — Advanced Workflow (DEEP)

> **Complexity:** DEEP  
> **When to use:** Architecture changes, migrations, complex integrations, security-critical features  
> **Active agents:** ALL agents participate

---

## How to Trigger

- "Migrate from monolith to microservices architecture"
- "Implement real-time WebSocket communication"
- "Add end-to-end encryption for messaging"
- "Refactor the entire state management to use Riverpod"

Or explicitly request: "Use DEEP pipeline for this"

## What Happens

All 10 agents are fully activated:

```
1. Orchestrator: Full triage, all agents evaluate
2. Researcher: Deep analysis, risk matrix, 4+ alternatives
3. Planner: Comprehensive plan with phases, rollback strategy
4. Builder: Enterprise-grade implementation, design patterns
5. UI/UX Specialist: Full design review if UI involved
6. Platform Guardian: All platforms verified
7. Validator: Unit + integration + e2e + security tests
8. Documentation: Full documentation suite
9. Project Manager: Complete telemetry, evolution tracking
10. Delivery: Full release checklist, version bump
```

## Key Differences from STANDARD

| Aspect | STANDARD | DEEP |
|--------|----------|------|
| Research depth | 3 alternatives | 4+ alternatives + risk matrix |
| Planning | Task list + criteria | Phases + rollback + dependencies |
| Testing | Unit + integration | Unit + integration + e2e + security |
| Documentation | Changelog + API docs | Full suite + architecture docs |
| Review | Quick check | Full design + platform review |

## When to Use DEEP vs CRITICAL

- **DEEP:** Architecture changes that DON'T involve regulated data or production deployment
- **CRITICAL:** Payment systems, health data, production deployments, compliance requirements
