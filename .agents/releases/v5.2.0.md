# HEPHAESTUS v5.2.0 — Integrity and Version Alignment

> Release type: Minor
> Date: 2026-04-23
> Scope: Framework integrity checks, metadata alignment, and path hygiene
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v5.2.0 makes the framework more verifiable without changing the core agent operating model.

The release adds a local integrity manifest and validation script, aligns version metadata across the active framework files, and fixes stale references that could confuse users or future maintainers.

## Added

- `.agents/config/framework-manifest.yaml`
  - declares expected framework version, core counts, required files, and required directories.
- `.agents/tools/validate-framework.ps1`
  - validates required files and directories;
  - validates expected counts for agents, protocols, workflows, memory stores, and documentation languages;
  - validates basic agent contracts (`AGENT.md` and `capabilities.yaml`);
  - validates local Markdown links;
  - reports missing `.agents/...` references as warnings by default.
- Context DB templates:
  - `.agents/memory/context-db/session-checkpoint.md`
  - `.agents/memory/context-db/session-brief.md`
  - `.agents/memory/context-db/next-chat-activation-prompt.md`
- Optional config override templates:
  - `.agents/config/project.yaml`
  - `.agents/config/multi-project.yaml`

## Changed

- Updated framework version metadata to `5.2.0` in active entry points.
- Updated `agent-registry.yaml` from `4.0.2` to `5.2.0`.
- Updated health dashboard expected values:
  - current version: `5.2.0`;
  - protocol count: `13`.
- Updated `smart-loading-protocol.md` references to existing prompt files.
- Replaced the stale `inter-agent-communication.md` reference with the active collaboration and handoff protocols.
- Updated setup guidance to use the v5.2.0 package and current framework counts.
- Converted example output paths in workflows to placeholders so they are not mistaken for required files.

## Preserved

- No agent behavior was removed.
- No broad technology catalog was restored.
- No default loading tier was expanded.
- The official documentation structure from v5.1.0 remains unchanged.

## Validation

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .agents/tools/validate-framework.ps1
```

For stricter path-reference checking:

```powershell
powershell -ExecutionPolicy Bypass -File .agents/tools/validate-framework.ps1 -StrictReferences
```

Current validation result for this release:

- `PASS=63`
- `FAIL=0`
- `WARN=0`

## Compatibility

The release is additive and should be safe for existing projects. It adds validation tooling and corrects metadata; it does not require users to change their daily workflow.
