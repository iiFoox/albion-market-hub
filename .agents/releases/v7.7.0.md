# HEPHAESTUS v7.7.0 — Optional Telemetry and Memory Proof

> Release Date: 2026-04-28
> Package: `HEPHAESTUS-Framework-v7.7.0.zip`
> Scope: Compact memory consultation proof and optional telemetry retention evidence

---

## Summary

v7.7.0 adds a compact, optional proof layer for memory consultation. The framework can now show that memory policy, index-first consultation, session-boundary memory behavior, telemetry schema support, and dry-run retention checks are wired correctly without loading full memory stores in normal work.

## Added

- `.agents/protocols/optional-telemetry-memory-proof-protocol.md`
  - defines optional proof behavior;
  - forbids broad memory-store loading for proof generation;
  - documents dry-run-first retention behavior.
- `.agents/tools/run-memory-proof.ps1`
  - generates compact memory proof evidence;
  - checks memory policy, protocol, index-first behavior, full-store default, schema, and retention dry-run.
- `.agents/tools/check-memory-proof.ps1`
  - validates protocol, tools, config, schema, Smart Loading, and pre-release gate wiring.
- `.agents/reports/memory/`
  - stores memory proof templates and generated compact reports.
- `.agents/docs/en/21-OPTIONAL-TELEMETRY-MEMORY-PROOF.md`
- `.agents/docs/pt-br/21-PROVA-OPCIONAL-DE-TELEMETRIA-E-MEMORIA.md`

## Changed

- Updated active framework metadata to `7.7.0`.
- Added `optional_telemetry_memory_proof` to active framework config.
- Added `proof` policy section to memory policy.
- Added `memory-proof` log type and `memory.proof_generated` event to telemetry schema.
- Added `memory-proof` Smart Loading group.
- Added memory proof checker and runner to the pre-release gate.
- Added memory proof evidence to the release evidence bundle.
- Updated bilingual documentation indexes, complete references, translation map, roadmap, changelog, and package checklist.

## Safety and Compatibility

- Normal sessions do not emit mandatory memory telemetry.
- Memory proof does not load full memory stores.
- Retention cleanup remains dry-run-first and archive-only when Apply is explicitly requested.
- No network access is required.
