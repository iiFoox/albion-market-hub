# HEPHAESTUS v6.1.0 — Safe Install and Update Automation

> Release type: Minor
> Date: 2026-04-27
> Scope: Portable install/update automation with dry-run safety and backups
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v6.1.0 makes the stable kernel portable across projects without weakening the safety model.

The release adds dry-run-first tools for comparing framework versions, installing HEPHAESTUS into a new project, and updating an existing project with backup and project-local config preservation.

## Added

- `.agents/tools/compare-framework-version.ps1`
  - compares source and target framework versions;
  - reports whether the source is newer, same, or older.
- `.agents/tools/install-framework.ps1`
  - installs `.agents` into a target project;
  - defaults to dry-run;
  - refuses to install over an existing `.agents` directory.
- `.agents/tools/update-framework.ps1`
  - updates an existing target `.agents`;
  - defaults to dry-run;
  - creates a backup ZIP by default when applying;
  - preserves project-local config and session context by default.
- `.agents/tools/test-installation-tools.ps1`
  - validates install/update tooling behavior in a temporary directory;
  - confirms dry-run does not copy files or create backups.
- `.agents/kernel/INSTALL-UPDATE.md`
  - documents safe install/update workflows.

## Changed

- Updated framework metadata to `6.1.0`.
- Added install/update tool paths to active config.
- Added install/update files to the integrity manifest.
- Added installation tooling validation to the pre-release gate.
- Updated kernel docs and complete references for v6.1.0.

## Preserved

- No automatic update is performed.
- Install and update remain manual commands.
- Dry-run is the default behavior.
- Project-local config and session state are preserved by default during updates.
- Normal framework loading behavior is unchanged.
