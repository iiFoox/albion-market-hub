## Summary

v7.0.0 hardens real-project execution without making real-project mutation permissive.

The framework now proves that controlled Apply is backup-first, approval-gated, audited, protected-path aware, and restorable in an isolated fixture.

## Added

- `.agents/protocols/real-project-execution-hardening-protocol.md`
  - defines Apply, backup, audit, protected-path override, and restore gates.
- `.agents/reports/executions/real-project-hardening-template.md`
  - documents hardening report expectations.
- `.agents/tools/check-real-project-hardening.ps1`
  - validates hardening protocol, templates, tool capabilities, and fixture proof.
- Backup artifact support in `.agents/tools/run-real-project-execution.ps1`.
- Audit trail support in `.agents/tools/run-real-project-execution.ps1`.
- Restore latest backup support in `.agents/tools/run-real-project-execution.ps1`.
- Controlled Apply fixture in `.agents/tools/test-real-project-execution.ps1`.

## Changed

- Updated framework metadata to `7.0.0`.
- Added `real_project_execution_hardening` to active framework execution config.
- Added `real-hardening` conditional Smart Loading group.
- Added hardening validation to the pre-release gate.
- Updated manifest expected protocol count to `19`.
- Updated EN/PT-BR complete references with v7.0.0.

## Preserved

- Default real-project execution remains `analysis_only`.
- Apply is still blocked by default in the real framework workspace.
- Hardening tests use an isolated temporary fixture.
- Normal LITE loading remains compact.
