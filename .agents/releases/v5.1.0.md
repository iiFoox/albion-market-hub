# HEPHAESTUS v5.1.0 — Bilingual Documentation Structure

> Release type: Minor
> Date: 2026-04-23
> Scope: Documentation organization and version alignment
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v5.1.0 introduces a complete bilingual documentation structure while keeping the operational framework core in English.

The framework still follows the established language policy:

- technical definitions, agent instructions, protocols, configuration, and memory stay in English;
- user communication may happen in Portuguese (pt-BR);
- user-facing documentation is now available in separate English and Portuguese folders.

## Added

- `.agents/docs/README.md` as the new documentation entry point.
- `.agents/docs/en/` for complete English user-facing documentation.
- `.agents/docs/pt-br/` for complete Brazilian Portuguese user-facing documentation.
- `.agents/docs/en/HEPHAESTUS-COMPLETE-REFERENCE.md` as the complete English reference.
- `.agents/docs/en/daily-prompts/` with English daily usage prompts.
- `.agents/docs/pt-br/daily-prompts/` with Portuguese daily usage prompts.
- `.agents/releases/v5.1.0.md` as the release note for this documentation restructuring.

## Changed

- Documentation navigation now points to `.agents/docs/en/` and `.agents/docs/pt-br/`.
- Framework documentation version is aligned to `5.1.0` for the bilingual documentation release.
- The legacy `.agents/Tutorial/` and `.agents/daily-prompts/` folders remain as README-only compatibility bridges.

## Preserved

- The operational source of truth remains `.agents/AGENTS.md`.
- Agent definitions remain in `.agents/agents/**`.
- Protocols remain in `.agents/protocols/**`.
- Workflows remain in `.agents/workflows/**`.
- Configuration remains in `.agents/config/**`.
- No agent behavior was removed.

## Compatibility

Existing references to the old documentation folders land on README bridge files. New documentation should use `.agents/docs/`.
