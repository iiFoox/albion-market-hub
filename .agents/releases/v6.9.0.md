# HEPHAESTUS v6.9.0 — Benchmark Expansion

> Release type: Minor
> Date: 2026-04-27
> Scope: Expanded isolated developer benchmark scenarios
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v6.9.0 expands the developer benchmark from four scenarios to ten.

The benchmark now tests broader work categories: UI feature, documentation update, bug fix, validation correction, API behavior, fake database migration, refactor, security fix, failing-test correction, and docs-only release.

## Added

- New developer benchmark scenarios:
  - `api`
  - `database-migration`
  - `refactor`
  - `security-fix`
  - `failing-tests`
  - `docs-only-release`
- Scenario presence validation in `.agents/tools/test-developer-benchmark.ps1`.
- Explicit `benchmark_scenarios` list in `.agents/config/framework.yaml`.

## Changed

- Updated `.agents/tools/run-developer-benchmark.ps1` to run ten scenarios by default.
- Updated benchmark report frontmatter to `framework_version: "6.9.0"`.
- Updated framework metadata to `6.9.0`.
- Updated EN/PT-BR complete references with the v6.9.0 release row.
- Updated roadmap and changelog for the expanded benchmark.

## Preserved

- Benchmark execution remains isolated in temporary workspaces.
- Temporary workspaces are removed by default.
- Normal Smart Loading behavior is unchanged.
- Package verification still skips report-generating benchmark execution.
