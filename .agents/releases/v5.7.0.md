# HEPHAESTUS v5.7.0 — Documentation Source of Truth and Reference Generation

> Release type: Minor
> Date: 2026-04-27
> Scope: Bilingual documentation parity and reference validation
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v5.7.0 prevents silent drift between English and Portuguese documentation.

The release adds an explicit translation map, a documentation source notes folder, and a local parity checker that validates paired docs, version headers, and key reference counts.

## Added

- `.agents/docs/_translation-map.yaml`
  - maps EN files to PT-BR equivalents;
  - records reference expectations for version, agents, protocols, workflows, memory stores, and languages.
- `.agents/docs/_source/README.md`
  - documents the documentation source-of-truth model.
- `.agents/tools/check-doc-parity.ps1`
  - validates mapped EN/PT-BR pairs exist;
  - detects unmapped Markdown files;
  - validates core documentation version headers;
  - validates complete references mention expected counts.

## Changed

- Updated framework metadata to `5.7.0`.
- Added documentation parity files to the integrity manifest.
- Updated complete references and documentation indexes to v5.7.0.
- Updated the health report metadata writer to emit `framework_version: "5.7.0"`.
- Aligned `ROADMAP.md` with completed v5.5.0, v5.6.0, and v5.7.0 work and added the v5.8.0 next step.

## Preserved

- No automatic translation was introduced.
- The operational framework core remains in English.
- The existing bilingual documentation structure remains unchanged.
- Token loading behavior is unchanged.
