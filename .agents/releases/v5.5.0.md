# HEPHAESTUS v5.5.0 — Telemetry Schema and Health Score

> Release type: Minor
> Date: 2026-04-26
> Scope: Telemetry schema, telemetry validation, and framework health scoring
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v5.5.0 turns telemetry into a verifiable contract and adds a local framework health score.

The release keeps telemetry lightweight: logs remain Markdown files, but new logs can now follow a simple YAML frontmatter schema. Legacy logs are supported as warnings by default.

## Added

- `.agents/config/telemetry-schema.yaml`
  - defines required frontmatter fields;
  - defines allowed log types;
  - defines allowed statuses;
  - lists canonical telemetry events.
- `.agents/tools/validate-telemetry.ps1`
  - validates telemetry log frontmatter;
  - reports legacy logs without frontmatter as warnings by default;
  - supports strict mode.
- `.agents/tools/framework-health.ps1`
  - runs framework integrity validation;
  - runs telemetry validation;
  - runs LITE loading estimate;
  - calculates a local health score;
  - can write `.agents/reports/health/latest.md`.
- `.agents/reports/health/README.md`

## Changed

- Updated framework metadata to `5.5.0`.
- Added telemetry schema and health report paths to `framework.yaml`.
- Added v5.5.0 files to the integrity manifest.
- Updated health dashboard guidance to reference the health script.

## Preserved

- Existing telemetry logs remain valid as legacy logs.
- Telemetry still uses Markdown files.
- No network or external service is required.
- Normal framework loading behavior is unchanged.
