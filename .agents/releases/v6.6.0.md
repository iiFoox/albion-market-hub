# HEPHAESTUS v6.6.0 — Project Discovery and Requirements Maturation

> Release type: Minor
> Date: 2026-04-27
> Scope: Conditional project discovery mode before implementation
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v6.6.0 adds a structured Project Discovery mode so HEPHAESTUS can mature product ideas before implementation.

This release addresses the common failure mode where an operator starts from a superficial idea and the system builds too early. The new mode asks deeper questions, challenges assumptions, compares directions, documents decisions, and checks readiness before planning code.

## Added

- `.agents/protocols/project-discovery-protocol.md`
  - defines activation triggers, discovery domains, output contract, readiness gate, and token policy.
- `.agents/config/project-discovery-policy.yaml`
  - configures conditional discovery behavior, required domains, cost control, documentation, and token policy.
- `.agents/workflows/project-discovery-workflow.md`
  - defines the agent flow for project discovery.
- `.agents/reports/discovery/README.md`
  - explains discovery report usage and token policy.
- `.agents/reports/discovery/project-discovery-template.md`
  - full living report template for product story, users, journeys, backend, data, costs, legal/IP, Git, risks, and readiness.
- `.agents/reports/discovery/decision-log-template.md`
  - compact template for direction-changing decisions.
- `.agents/tools/check-project-discovery.ps1`
  - validates discovery protocol, policy, workflow, templates, docs, loading group, and translation map.
- `.agents/docs/en/12-PROJECT-DISCOVERY.md`
- `.agents/docs/pt-br/12-DESCOBERTA-DE-PROJETO.md`

## Changed

- Updated framework metadata to `6.6.0`.
- Added `project_discovery` to active framework execution config.
- Added the `discovery` conditional group to Smart Loading.
- Added Project Discovery validation to the pre-release gate.
- Updated the integrity manifest to expect `16` protocols and `10` workflows.
- Updated EN/PT-BR documentation map and complete references.
- Updated roadmap and changelog for the new project maturation capability.

## Preserved

- Project Discovery is conditional and does not load during normal LITE tasks.
- The framework still preserves Smart Loading and token economy.
- Implementation is still gated by operator confirmation after discovery readiness.
- Existing v6.5 documentation enforcement remains active.
