# HEPHAESTUS v6.8.0 — Developer Execution on Real Local Project

> Release type: Minor
> Date: 2026-04-27
> Scope: DryRun-first controlled execution planning for real local projects
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v6.8.0 adds the first controlled real-project execution layer.

It does not let the framework mutate real projects freely. Instead, it introduces a DryRun-first workflow that validates the Real Project Adapter, produces an execution plan, checks protected paths, requires backup and approval, and writes a real-project execution report.

## Added

- `.agents/protocols/developer-real-project-execution-protocol.md`
  - defines DryRun-first execution, Apply preconditions, approval gates, protected-path checks, and output contract.
- `.agents/workflows/real-project-execution-workflow.md`
  - defines the agent flow for controlled real-project execution.
- `.agents/reports/executions/real-project-execution-plan-template.md`
  - template for real-project execution status reports.
- `.agents/tools/run-real-project-execution.ps1`
  - reads `.agents/config/project.yaml`, validates safety gates, generates DryRun status, and blocks unsafe Apply attempts.
- `.agents/tools/test-real-project-execution.ps1`
  - validates DryRun behavior, Apply blocking, and report frontmatter.
- `.agents/tools/check-real-project-execution.ps1`
  - validates protocol, workflow, tools, docs, project safety config, loading group, and translation map.
- `.agents/docs/en/14-REAL-PROJECT-EXECUTION.md`
- `.agents/docs/pt-br/14-EXECUCAO-EM-PROJETO-REAL.md`

## Changed

- Updated framework metadata to `6.8.0`.
- Added `real_project_execution` to active framework execution config.
- Added the `real-execution` conditional group to Smart Loading.
- Added real-project execution checks and tests to the pre-release gate.
- Updated the integrity manifest to expect `18` protocols and `12` workflows.
- Updated EN/PT-BR documentation map and complete references.
- Updated roadmap and changelog for controlled real-project execution.

## Preserved

- Default project execution mode remains `analysis_only`.
- Apply is blocked unless the adapter is in `controlled` mode and approval is explicit.
- Destructive commands remain disabled.
- Protected paths are enforced.
- Real-project execution context is conditional and does not load during normal LITE tasks.
