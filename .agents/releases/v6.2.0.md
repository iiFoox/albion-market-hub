# HEPHAESTUS v6.2.0 — Isolated Simulation Harness

> Release type: Minor
> Date: 2026-04-27
> Scope: Temporary-workspace framework simulation and behavior reporting
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v6.2.0 adds an isolated simulation harness so HEPHAESTUS can test itself acting in a disposable project workspace without polluting the framework directory.

The harness installs the framework into a temporary project, creates a simulated request from a routing fixture, runs routing/loading/gate checks, writes a compact report, and deletes the temporary workspace by default.

## Added

- `.agents/tools/run-framework-simulation.ps1`
  - creates an isolated temporary workspace;
  - installs the framework into a simulated project;
  - writes a scenario request file in the isolated workspace;
  - runs version comparison, routing fixtures, loading estimate, and pre-release gate;
  - removes the workspace by default unless `-KeepWorkspace` is used.
- `.agents/tools/test-simulation-harness.ps1`
  - validates the simulation harness;
  - confirms a report is generated with schema-compatible frontmatter.
- `.agents/reports/simulations/README.md`
  - documents generated simulation reports.

## Changed

- Updated framework metadata to `6.2.0`.
- Added simulation config to active framework configuration.
- Added simulation tools and reports directory to the integrity manifest.
- Added simulation harness validation to the pre-release gate.
- Updated telemetry schema to allow simulation reports.
- Updated EN/PT-BR complete references with the v6.2.0 release row.

## Preserved

- Simulation does not write into framework internals except for the optional compact report.
- Temporary workspaces are removed by default.
- Normal framework loading behavior is unchanged.
- Install/update safety rules remain unchanged.
