# HEPHAESTUS v6.4.0 — Multi-Scenario Developer Benchmark

> Release type: Minor
> Date: 2026-04-27
> Scope: Repeated developer execution across multiple controlled scenarios
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v6.4.0 expands developer execution from one controlled scenario into a repeatable benchmark.

The benchmark runs multiple isolated scenarios, validates each mini project, checks the installed framework gate, writes a consolidated benchmark report, and removes temporary workspaces by default.

## Added

- `.agents/tools/run-developer-benchmark.ps1`
  - runs developer scenarios in isolated temporary workspaces;
  - covers `profile-card`, `docs-update`, `bug-fix`, and `validation-failure`;
  - installs the framework into each project;
  - runs local scenario tests;
  - validates the installed framework gate.
- `.agents/tools/test-developer-benchmark.ps1`
  - validates the benchmark runner;
  - verifies benchmark report frontmatter.
- `.agents/reports/executions/benchmark-latest.md`
  - generated consolidated benchmark report.

## Changed

- Updated framework metadata to `6.4.0`.
- Added benchmark paths to active `developer_execution` config.
- Added benchmark tools to the integrity manifest.
- Added benchmark validation to the pre-release gate before packaging.
- Updated telemetry schema to allow developer benchmark reports.
- Updated EN/PT-BR complete references with the v6.4.0 release row.

## Preserved

- Benchmark scenarios do not modify the real project.
- Temporary workspaces are removed by default.
- Package verification skips report-generating execution checks.
- Normal framework loading behavior is unchanged.
