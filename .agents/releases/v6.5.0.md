# HEPHAESTUS v6.5.0 — Documentation Enforcement and Mastery Calibration

> Release type: Minor
> Date: 2026-04-27
> Scope: Continuous documentation checkpoint and master-level agent calibration
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v6.5.0 closes two important maturity gaps without increasing normal token cost: Documentation is now an enforced lightweight checkpoint, and every agent is calibrated to Master-level behavior with 30+ years equivalent expertise.

This release does not add heavy runtime behavior. It adds governance, validation, and clearer agent standards so the framework keeps documentation aligned while still preserving Smart Loading and minimal-context execution.

## Added

- `.agents/protocols/documentation-enforcement-protocol.md`
  - defines when Documentation must act;
  - separates full documentation work from lightweight impact checks;
  - allows `not_impacted` only with a recorded reason.
- `.agents/config/documentation-policy.yaml`
  - establishes the active documentation policy;
  - keeps documentation enforcement token-aware;
  - defines required documentation impact outcomes.
- `.agents/reports/documentation/README.md`
  - describes documentation reports and impact notes.
- `.agents/reports/documentation/documentation-impact-template.md`
  - provides a compact template for documentation decisions.
- `.agents/tools/check-documentation-enforcement.ps1`
  - validates Documentation activation, policy, template, protocol, and pre-release integration.
- `.agents/tools/check-agent-mastery.ps1`
  - verifies all agent specs include Master-level and 30+ years equivalent calibration.

## Changed

- Updated framework metadata to `6.5.0`.
- Set Documentation as always active in `.agents/config/agent-registry.yaml`.
- Made Documentation mandatory in the registry pipeline.
- Added `documentation_enforcement` to `.agents/config/framework.yaml`.
- Added agent mastery rules to the framework principles.
- Added documentation enforcement and agent mastery checks to the pre-release gate.
- Updated all agent specifications with Master-level / 30+ years equivalent calibration.
- Updated system prompts to match the new mastery standard.
- Updated kernel guarantees to include documentation enforcement and master-level calibration.
- Updated EN/PT-BR complete references with the v6.5.0 release row.

## Preserved

- Normal Smart Loading behavior remains unchanged.
- Documentation enforcement uses a lightweight checkpoint by default.
- Full documentation generation is only required when the change actually affects documented behavior, architecture, usage, releases, or decisions.
- Existing simulation, developer execution, benchmark, install/update, kernel, telemetry, routing, and parity gates remain intact.
