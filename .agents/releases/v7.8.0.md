# HEPHAESTUS v7.8.0 — Final Operator Experience Review

> Release Date: 2026-04-28
> Package: `HEPHAESTUS-Framework-v7.8.0.zip`
> Scope: Operator-facing navigation, stale prompt cleanup, and final experience review

---

## Summary

v7.8.0 improves the human operating experience around HEPHAESTUS. It adds an intent-based operator map, updates stale ready prompts and legacy bridge files, and introduces a lightweight checker that prevents old entry points from drifting behind the current framework version.

## Added

- `.agents/protocols/final-operator-experience-review-protocol.md`
  - defines the operator experience review scope and token boundary.
- `.agents/docs/en/22-OPERATOR-EXPERIENCE-MAP.md`
  - maps operator intent to the right English guide.
- `.agents/docs/pt-br/22-MAPA-DE-EXPERIENCIA-DO-OPERADOR.md`
  - maps operator intent to the right Portuguese guide.
- `.agents/tools/run-operator-experience-review.ps1`
  - generates a compact operator experience report.
- `.agents/tools/check-operator-experience.ps1`
  - validates the operator map, ready prompts, legacy bridges, config, loading group, and gate wiring.
- `.agents/reports/operator/`
  - stores operator experience templates and generated reports.

## Changed

- Updated active framework metadata to `7.8.0`.
- Added `operator_experience_review` to active framework config.
- Added `operator-experience` Smart Loading group.
- Updated ready prompts in EN/PT-BR to v7.8.0.
- Updated legacy `.agents/Tutorial/README.md` and `.agents/daily-prompts/README.md` bridge versions.
- Added operator experience evidence to the release evidence bundle.
- Updated bilingual docs indexes, complete references, translation map, roadmap, changelog, and package checklist.

## Safety and Compatibility

- No execution behavior changed.
- No project mutation behavior changed.
- The operator map reduces token usage by helping the host load only the relevant guide.
- Legacy folders remain compatibility bridges.
