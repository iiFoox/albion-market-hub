# HEPHAESTUS v5.8.0 — Practical Enforcement Layer

> Release type: Minor
> Date: 2026-04-27
> Scope: Pre-release enforcement, package verification, and optional telemetry cleanup
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v5.8.0 turns the existing validation tools into a practical release gate.

The framework now has a single pre-release command that runs integrity checks, documentation parity, telemetry validation, routing fixtures, health reporting, and loading estimates. It can also verify a generated ZIP against the integrity manifest.

## Added

- `.agents/tools/pre-release-gate.ps1`
  - runs the full local release validation sequence;
  - supports `-Version`;
  - supports optional `-PackagePath` ZIP verification;
  - avoids rewriting the health report during package verification;
  - supports `-StrictTelemetry` when legacy telemetry has been cleaned.
- `.agents/tools/cleanup-telemetry.ps1`
  - identifies telemetry logs older than a retention threshold;
  - defaults to dry-run mode;
  - archives old logs only when explicitly called with `-Apply -Archive`.
- `.agents/releases/PACKAGE-CHECKLIST.md`
  - documents the release validation and ZIP verification workflow.

## Changed

- Updated framework metadata to `5.8.0`.
- Added release governance paths to active config because they now point to real tooling.
- Added telemetry cleanup tool path to active config with manual cleanup mode.
- Added v5.8.0 tooling and checklist to the integrity manifest.
- Updated `framework-health.ps1` so health reports read the active framework version from `.agents/config/framework.yaml`.
- Updated EN/PT-BR complete references with the v5.8.0 release row.

## Preserved

- No always-on background job was introduced.
- Telemetry cleanup is manual and dry-run by default.
- Normal agent loading behavior is unchanged.
- Existing documentation parity and routing contracts remain intact.
