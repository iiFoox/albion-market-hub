# HEPHAESTUS v5.4.0 — Memory Contract and Session State

> Release type: Minor
> Date: 2026-04-26
> Scope: Triggered memory consultation, memory policy, and session state templates
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v5.4.0 makes memory operational without making it heavy.

The release adds a triggered, tier-aware memory consultation protocol, a compact memory policy, and reusable templates for durable memory entries. The goal is to preserve continuity and learning while protecting the Smart Loading budget.

## Added

- `.agents/protocols/memory-consultation-protocol.md`
  - defines when memory must be consulted;
  - defines tier-aware memory budgets;
  - defines read order;
  - defines write-back rules;
  - defines compact consultation notes.
- `.agents/config/memory-policy.yaml`
  - configures triggered consultation;
  - defines per-tier memory file budgets;
  - points to reusable memory templates.
- `.agents/memory/templates/`
  - `decision.md`
  - `learning.md`
  - `risk.md`
  - `project-state.md`
  - `next-action.md`

## Changed

- Updated framework metadata to `5.4.0`.
- Updated protocol count from 13 to 14.
- Added memory consultation settings to `framework.yaml`.
- Added `memory-policy.yaml` to the Smart Loading core files.
- Updated `MEMORY.md` to point to the consultation protocol and policy.
- Updated `MEMORY-INDEX.md` with session state files and memory templates.
- Updated `ROADMAP.md` so the old v4.x memory item now reflects the implemented v5.4.0 memory contract.

## Preserved

- Memory is not loaded wholesale by default.
- LITE and STANDARD tiers remain compact.
- Memory consultation is triggered and tier-aware.
- Existing memory governance remains valid.
