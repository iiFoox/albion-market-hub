# HEPHAESTUS v6.3.0 — Developer Execution Mode Simulation

> Release type: Minor
> Date: 2026-04-27
> Scope: Controlled developer execution inside disposable mini projects
> Changelog: [CHANGELOG.md](CHANGELOG.md)

---

## Summary

v6.3.0 adds the first controlled simulation of HEPHAESTUS acting as a developer.

The execution harness creates an isolated mini project, installs the framework, creates a profile-card feature, writes HTML/CSS/JavaScript, writes a local structural test, runs that test, validates the installed framework, writes a delivery note, and generates a compact execution report. The workspace is deleted by default.

## Added

- `.agents/tools/run-developer-execution-simulation.ps1`
  - creates an isolated developer workspace;
  - installs the framework;
  - writes a controlled implementation plan;
  - creates a small responsive profile-card app;
  - adds a local structural test;
  - writes a delivery note;
  - validates the installed framework gate;
  - removes the workspace by default unless `-KeepWorkspace` is used.
- `.agents/tools/test-developer-execution-mode.ps1`
  - validates the developer execution harness;
  - verifies the execution report frontmatter.
- `.agents/reports/executions/README.md`
  - documents generated developer execution reports.

## Changed

- Updated framework metadata to `6.3.0`.
- Added `developer_execution` to active config as controlled simulation only.
- Added developer execution tools and reports directory to the integrity manifest.
- Added developer execution validation to the pre-release gate before packaging.
- Updated telemetry schema to allow developer execution reports.
- Updated EN/PT-BR complete references with the v6.3.0 release row.

## Preserved

- Developer execution does not modify the real project.
- The mini project is disposable.
- Temporary workspaces are removed by default.
- Package verification skips report-generating execution checks so final ZIP validation does not mutate reports.
- Normal framework loading behavior is unchanged.
