# HEPHAESTUS v8.2.0 — Operator Daily Launcher and First-Call Installer

> Release Date: 2026-04-28
> Package: `HEPHAESTUS-Framework-v8.2.0.zip`
> Scope: Daily operator simplification and Windows first-install launcher

---

## Summary

v8.2.0 adds low-friction operator entry points for daily work, validation, release preparation, and first installation into target projects.

## Added

- `.agents/tools/operator-daily.ps1`
- `.agents/tools/install-hephaestus-launcher.ps1`
- `.agents/tools/install-hephaestus.bat`
- `.agents/tools/check-operator-daily-launcher.ps1`
- `.agents/protocols/operator-daily-launcher-protocol.md`
- `.agents/docs/en/26-OPERATOR-DAILY-LAUNCHER.md`
- `.agents/docs/pt-br/26-LANCADOR-DIARIO-DO-OPERADOR.md`

## Changed

- Added `daily` to `.agents/tools/hephaestus.ps1`.
- Added `operator_daily_launcher` to active framework config.
- Added `operator-daily` to Smart Loading conditional groups.
- Integrated the checker into the pre-release gate and doctor.
- Updated manifest, translation map, roadmap, changelog, and complete references.

## Safety and Compatibility

- Install remains dry-run unless Apply is explicit.
- The `.bat` launcher is a transparent wrapper over PowerShell, not a new opaque binary.
- The launcher does not delete target files.
- Existing `.agents` targets still require update instead of install.
- A generated `HEPHAESTUS-FIRST-CALL.md` is created only after Apply succeeds.

## Validation

Expected release validation:

- `check-operator-daily-launcher.ps1`
- `hephaestus.ps1 -Action daily -DailyMode start`
- `hephaestus.ps1 -Action validate`
- `check-doc-parity.ps1`
- `validate-framework.ps1 -StrictReferences`
- `hephaestus.ps1 -Action gate -Version 8.2.0`
- Package gate with `HEPHAESTUS-Framework-v8.2.0.zip`
