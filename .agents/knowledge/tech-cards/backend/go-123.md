# Go 1.23 — Tech Card

> **Category:** Backend Language
> **Current Version:** 1.23.x
> **Type:** Compiled, statically typed, concurrent

---

## Quick Setup
```bash
go mod init myapp
go get github.com/gin-gonic/gin  # Popular web framework
```

## Key Features
- **Goroutines** — lightweight concurrent execution (thousands of concurrent tasks)
- **Channels** — safe communication between goroutines
- **Fast compilation** — compiles in seconds, not minutes
- **Single binary** — no runtime dependencies, easy deployment
- **Built-in tooling** — `go test`, `go vet`, `go fmt`, `go doc`
- **Generics** — type parameters for reusable functions/types (since 1.18)

## Top 10 Best Practices

1. **Use `go fmt`** — standardized formatting (non-negotiable)
2. **Handle errors explicitly** — `if err != nil { return err }`
3. **Use interfaces for abstraction** — small interfaces (1-3 methods)
4. **Use goroutines + channels** — not mutexes (share by communicating)
5. **Use `context.Context`** — pass to all functions for cancellation/timeout
6. **Use `defer`** — for cleanup (close files, unlock mutexes)
7. **Use table-driven tests** — standard Go testing pattern
8. **Use `errgroup`** — for grouped goroutine management with error handling
9. **Use `slog`** — structured logging (standard library since 1.21)
10. **Use `golangci-lint`** — comprehensive linting (replaces multiple linters)

## Top 10 Gotchas

1. ❌ **Goroutine leak** — always ensure goroutines can exit (use `context.Done()`)
2. ❌ **Nil pointer on interface** — interface with nil value is NOT nil itself
3. ❌ **Loop variable capture** — fixed in Go 1.22+ but beware in older versions
4. ❌ **Ignoring errors** — `result, _ := fn()` silently hides bugs
5. ❌ **Unbuffered channel deadlock** — sender blocks until receiver is ready
6. ❌ **Slice append gotcha** — `append` may or may not create new backing array
7. ❌ **Map iteration order** — random by design, don't depend on it
8. ❌ **No constructors** — use `NewMyStruct()` factory functions
9. ❌ **Exported vs unexported** — capital letter = public, lowercase = private
10. ❌ **JSON tags** — struct fields must be exported AND tagged for JSON marshaling

## Error Handling Pattern
```go
// Define sentinel errors
var (
    ErrNotFound     = errors.New("not found")
    ErrUnauthorized = errors.New("unauthorized")
)

// Wrap errors with context
func (s *UserService) GetByID(ctx context.Context, id string) (*User, error) {
    user, err := s.repo.FindByID(ctx, id)
    if err != nil {
        return nil, fmt.Errorf("UserService.GetByID(%s): %w", id, err)
    }
    if user == nil {
        return nil, ErrNotFound
    }
    return user, nil
}

// Check errors with errors.Is()
if errors.Is(err, ErrNotFound) {
    http.Error(w, "User not found", http.StatusNotFound)
}
```

## Performance Tips
- Use `sync.Pool` for frequently allocated objects
- Use `bytes.Buffer` instead of string concatenation
- Profile with `go tool pprof` (CPU, memory, goroutines)
- Use `GOMAXPROCS` to control CPU parallelism
- Compile with `go build -ldflags="-s -w"` for smaller binaries
